## 1. 不可违反的前提：任何修改必须可回滚

| 操作 | 正确做法 | 违规做法 |
|------|---------|---------| 
| 替换文件 | 新文件放 `new/` 目录，验证后才删旧文件 | 直接覆盖原文件 |
| 修改代码 | 先 `git commit` 当前状态，再改 | 改完才发现没备份 |
| 删除文件 | 先移到临时目录，确认无用后再删 | 直接 rm / Remove-Item |
| 数据库变更 | 先备份数据，再执行迁移 | 直接删表/重建 |
| 依赖升级 | 先锁版本号，再升级 | 直接 npm install latest |

## 2. 技能路由（强制检查点）

| 时刻 | 技能 | 触发条件 |

## 4. 验证闭环

**核心规则：agent 报告不可信，必须独立验证。**

| 验证方式 | 适用场景 |
|----------|----------|
| 编译验证 | 任何代码修改 → `npm run build` |
| 测试验证 | 功能实现/修复 → `npm run test` |
| grep 验证 | 关键内容修改 → `grep -n "关键字" 文件` |
| 浏览器验证 | UI/交互修改 → Playwright 或手动验证 |

## 5. Git 工作流

```
1. git status --short        → 检查暂存区
2. 确认无运行时文件          → .db, .output/, dist/ 等
3. git diff --stat           → 确认改动范围合理
4. git commit -m "类型: 描述"
```
